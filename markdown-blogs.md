# Markdown-based Blogs

> [!note]
> Short version for now -- full guide coming.

A blog is the most naturally markdown-shaped thing there is: a folder of posts, one file each, a date and title in the frontmatter, newest first. Every blogging platform is essentially this plus lock-in.

**The pattern**: `blog/2026-07-18-my-post.md` with frontmatter (`title`, `date`, maybe `tags`). A publisher renders the folder as a blog listing and each file as a post.

**Tools**: write in anything ([Obsidian](markdown-knowledge-bases.md) is lovely for this). Publish with Flowershow ([blog listing in 30 seconds](https://www.youtube.com/watch?v=0NGvcuXTris)), or Hugo/Jekyll/Astro/Eleventy -- all of them eat markdown folders.

**Why**: your archive is yours. Platforms come and go (pour one out for every blog lost to a shutdown); a folder of markdown moves with you, forever. And AI tools can [work directly on your drafts](markdown-in-chatgpt.md).

Start with [markdown-based websites](markdown-websites.md) -- a blog is that plus dates.
