# Markdown-based Digital Gardens

> [!note]
> Short version for now -- full guide coming.

A digital garden is a public notebook -- notes published while still growing, linked into a web you wander rather than a feed you scroll. Less "blog post, final, timestamped", more "here's my thinking so far".

**The pattern**: it's a [knowledge base](markdown-knowledge-bases.md) with a public face. Keep notes in markdown with wiki-links; mark what's public; publish that slice. Readers follow links; backlinks make the garden explorable.

**Tools**: Obsidian for tending; Flowershow for publishing (it renders wiki-links and backlinks properly, which gardens need); GitHub in between.

The MOGF [tutorial series](learn/index.md) gets you a published garden by the end of tutorial 2.
