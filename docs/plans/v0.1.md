# Plan: v0.1 — a polished Way of Markdown site

Created 2026-07-18. Status: **in progress**.

## Vision (short)

The Way of Markdown is a practical guide with a philosophical spine: it teaches people — non-technical but sophisticated — how to build websites, docs, knowledgebases and more with markdown (esp. the Markdown + Obsidian + GitHub + Flowershow stack), and makes the case for *why* the markdown-based approach wins: openness, portability, simplicity, future-proofness, AI-friendliness. Craft ethos, not manifesto. See [[../naming|naming decision]].

## v0.1 goals

1. **Decent-looking site** — polished landing page (`index.md`, HTML/Tailwind where useful), clean nav, consistent config, renamed to "The Way of Markdown".
2. **Clear information architecture** — content sections agreed and reflected in nav + index pages (see below).
3. **All existing material integrated** — content migrated back from `flowershow` repo (`content/flowershow-app/markdown/`), merged with what's here, plus other relevant published content (other flowershow-repo notes; other repos TBD — Rufus to list).
4. **v0.2 sketch** — a short plan for what comes next (see bottom).

## Information architecture

Content sections (what a visitor sees). Shaped by the core insight (see [[../vision]]): markdown basics are trivial and not the point — the real curriculum is the markdown-based ecosystem and the use cases it unlocks (reproduce Notion/blogs/wikis/catalogs without lock-in).

- **Home** — the pitch: what/why in brief, routes into everything else.
- **Why** — the philosophical spine: why markdown-based, the manifesto, long-form pieces (e.g. The Markdown Database Pattern).
- **Learn** — two tracks: (a) *markdown basics* — the syntax core, quick; (b) *the markdown-based way* — use-case-driven: build a website (MOGF tutorials 1–4, the current arc), then blog / knowledgebase / catalog / database…
- **Guides** (how-tos) — task-oriented: markdown in Google Docs / Apple Notes / iPad, publish an Obsidian vault, AI-edit your vault, add images, GitHub tips…
- **Markdown Everywhere** (showcase) — how many tools now speak markdown; ubiquity as argument. Seeded from the kb "apps" notes; can start as one page.
- **Reference** — syntax notes (heading identifiers…), tools directory (into.md, Turndown, Markdoc…).

For v0.1, Guides + Everywhere + Reference can share one kb-style area with a curated index; nav stays simple (Learn / Knowledge Base / Why).

Directories: keep them **shallow** — sections are curation (index pages + navbar), not deep trees. `learn/` stays a folder (ordered series + assets); guides/reference notes can live flat or in one `kb/`-style folder — the sidebar grouping is the only real reason folders matter in Flowershow. Easy to change later; decide finally during migration.

Non-site files:

- `README.md` — repo-facing readme (not published); site landing moves to `index.md`.
- `docs/` — dev/meta docs: naming, plans, work log. All `publish: false`.
