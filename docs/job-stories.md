# Job stories

Who we're building for, in job-story form. The canonical list — issues and content decisions should reference these. Personas/marketing view: [[plans/marketing]] §2.

## JS1: The Life Itself new joiner (the founding story) ⭐

> When I join Life Itself (or any markdown-based team) knowing little or no markdown, git or GitHub, I want a single step-by-step path I can work through autonomously, so that within a few days I can edit the website, contribute to the knowledge base, and collaborate via PRs **without needing dev support**.

This is the story the whole project grew from (old epics #1 and #12). Current answer: [learn/onboarding](https://wayofmarkdown.com/learn/onboarding) + the roadmap page. Test of success: a real newcomer gets from zero to a merged PR with no human hand-holding.

## JS2: The searcher

> When I hit a markdown question in an app I already use ("how do I bold in WhatsApp?", "can Google Docs export markdown?"), I want a page that answers in the first screen, so I can get back to what I was doing.

Answer: the [everywhere](https://wayofmarkdown.com/everywhere) spokes, syntax table up top.

## JS3: The escaper

> When I've grown wary of Notion/Evernote/my CMS owning my content, I want a credible, honest path out — including what I'll lose — so I can move to files I own without regretting it.

Answer: [notion-alternative](https://wayofmarkdown.com/markdown-notion-alternative) and the guides section.

## JS4: The builder-curious

> When I want a website/blog/knowledge base but don't code and don't want a platform subscription, I want to be walked from "what even is this" to a published thing, so I can have it live this week.

Answer: the roadmap page → tutorial spine → [markdown-based](https://wayofmarkdown.com/markdown-based) guides.

## JS5: The team lead

> When my team's knowledge is scattered across tools, I want a curriculum and a pattern I can point people at, so the team converges on a workflow we own.

Answer: [wikis guide](https://wayofmarkdown.com/markdown-wikis) + onboarding path (JS1 generalised).

## Using these

- New content should serve a named story; if it serves none, question it.
- The roadmap/spine serves JS1 and JS4 (same journey, different entry motivation).
- JS2 pages exist to catch traffic and route a slice of it into JS3/JS4 journeys.
