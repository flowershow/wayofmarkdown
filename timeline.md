# The Markdown Timeline

How a 2004 Perl script for bloggers became the default format of the internet. (Primary sources for most of this: [the canon](kb/canon.md).)

<div style="border-left:3px solid #22c55e; padding-left:1.5rem; margin:2rem 0;">

<div style="margin-bottom:1.6rem;"><strong>1970s–90s · plain-text prehistory.</strong> Email and Usenet culture invents the conventions markdown would formalise: *asterisks* for emphasis, indentation for quoting, dashes for lists. People were writing proto-markdown before it had a name.</div>

<div style="margin-bottom:1.6rem;"><strong>1991–2002 · the ancestors.</strong> setext brings structured plain text to email newsletters; Aaron Swartz's <a href="http://www.aaronsw.com/2002/atx/intro">atx</a> (2002) contributes the <code>#</code> heading. Textile and reStructuredText explore the same territory.</div>

<div style="margin-bottom:1.6rem;"><strong>2004 · Markdown.</strong> John Gruber, with Swartz as collaborator and first user, releases <a href="https://daringfireball.net/projects/markdown/">Markdown</a>: a syntax and a Perl script, guided by one principle: the source should be readable as-is. The spec fits on one page. This turns out to matter more than any feature.</div>

<div style="margin-bottom:1.6rem;"><strong>2005–2012 · the web adopts it.</strong> Reddit, then GitHub and Stack Overflow (both 2008), make markdown the way millions write daily — READMEs, comments, questions. GitHub's dialect (GFM) adds tables and task lists and becomes the working standard. Markdown stops being a tool and becomes a habit.</div>

<div style="margin-bottom:1.6rem;"><strong>2014 · the standardisation drama.</strong> A decade of ambiguity ("is this a list?") prompts <a href="https://blog.codinghorror.com/standard-flavored-markdown/">Standard Flavored Markdown</a> — renamed <a href="https://commonmark.org/">CommonMark</a> after Gruber objects. John MacFarlane's rigorous spec becomes the foundation most implementations build on. "Simple" turns out to take hundreds of pages to pin down; users notice nothing, which is the point.</div>

<div style="margin-bottom:1.6rem;"><strong>2015–2020 · the tools era.</strong> Static site generators (Jekyll, Hugo) make markdown folders into websites. Notion speaks it at the edges; Bear, Typora and iA Writer polish the writing experience; <strong>Obsidian</strong> (2020) proves a serious knowledge tool can be <em>just markdown files</em>, and win precisely because of it.</div>

<div style="margin-bottom:1.6rem;"><strong>2022 · AI speaks markdown.</strong> ChatGPT launches and answers in markdown; every AI since does too. Trained on a markdown web and needing a token-cheap structure, <a href="markdown-in-chatgpt.md">AI makes markdown the native format of the era</a>, arguably the biggest boost since GitHub.</div>

<div style="margin-bottom:1.6rem;"><strong>2024–now · convergence.</strong> Google Docs adds real markdown import/export. WhatsApp expands its formatting. iPad Notes breaks Apple's holdout streak. <code>llms.txt</code> proposes markdown as the web's AI-facing layer. The world is <a href="everywhere.md">converging on markdown</a>, mostly without announcements.</div>

</div>

**The pattern across two decades**: markdown wins not by adding features but by being the smallest thing everyone can agree on: [readable, portable, pluggable](pluggable-markdown.md), and owned by no one. Where it goes next: [the manifesto](manifesto.md) has our bet.
