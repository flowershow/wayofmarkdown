# The Markdown Roadmap 🗺️

Everything on this site, as one path. Start at your entrance, follow the stages in order, branch at the end into whatever you want to build. Each stage links to the guide or tutorial that covers it; most stages are minutes, not days.

## Pick your entrance

<div class="grid md:grid-cols-2 gap-4 my-6">
  <a href="/learn/tutorials/create-a-website-from-scratch" class="block border-2 rounded-xl p-5 no-underline hover:shadow-lg transition-shadow">
    <div class="text-2xl mb-1">🚀 "I want to publish something"</div>
    <div>Website, blog, page on the web. Start at Stage 1, and you'll have something live by Stage 2 — today.</div>
  </a>
  <a href="/markdown-knowledge-bases" class="block border-2 rounded-xl p-5 no-underline hover:shadow-lg transition-shadow">
    <div class="text-2xl mb-1">📝 "I want better notes"</div>
    <div>Notes, knowledge base, second brain — maybe published later, maybe never. Start local at Stage 1, publishing waits until Stage 5 interests you.</div>
  </a>
</div>

Both entrances walk the same spine. The only difference is when you care about the publishing steps.

## The spine

<div class="border-l-4 pl-6 my-8 space-y-8">

<div>
  <div class="text-xl font-bold">⓪ &nbsp;Why bother? <span class="font-normal text-sm">(optional, 5 min)</span></div>
  <p>Understand the point before the mechanics: <a href="/why">why markdown-based</a>, or the punchier <a href="/manifesto">manifesto</a>. Skip if you're already convinced.</p>
</div>

<div>
  <div class="text-xl font-bold">① &nbsp;Learn markdown itself <span class="font-normal text-sm">(10–20 min)</span></div>
  <p>The syntax is genuinely small: <a href="/basics">Markdown Basics</a>, plus an <a href="https://commonmark.org/help/tutorial/">interactive tutorial</a> if you like exercises. You're done when headings, bold, links and lists come out of your fingers.</p>
</div>

<div>
  <div class="text-xl font-bold">② &nbsp;Publish your first page <span class="font-normal text-sm">(20 min)</span></div>
  <p>The "GitHub direct method": everything in the browser, nothing installed. <a href="/learn/tutorials/create-a-website-from-scratch">Tutorial 1</a> takes you from a free GitHub account to a live website 🎉 Notes-first people: fine to skim this and come back.</p>
</div>

<div>
  <div class="text-xl font-bold">③ &nbsp;Go local: Obsidian + GitHub Desktop <span class="font-normal text-sm">(40 min)</span></div>
  <p>The comfortable daily setup — files on your machine, a lovely editor, sync when ready. <a href="/learn/tutorials/edit-a-website-locally">Tutorial 2</a>, plus <a href="/markdown-in-obsidian">what makes Obsidian special</a>.</p>
</div>

<div>
  <div class="text-xl font-bold">④ &nbsp;Structure your stuff <span class="font-normal text-sm">(30 min)</span></div>
  <p>Frontmatter and wiki-links turn notes into <em>systems</em>: <a href="/learn/howtos/create-a-simple-catalog-of-anything">build a catalog of anything</a>, then meet <a href="/markdown-database">the markdown database pattern</a> behind it.</p>
</div>

<div>
  <div class="text-xl font-bold">⑤ &nbsp;Work with others <span class="font-normal text-sm">(20 min)</span></div>
  <p>Branches, pull requests, review: <a href="/learn/tutorial-3">tutorial 3</a> and the <a href="/kb/github-tips">GitHub tips</a>. Solo? Skip until a collaborator shows up.</p>
</div>

</div>
